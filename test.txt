1. перейти в директорию можно введя: cd "название папки"
2. ~ - корневая папка в ветке, . - текущая папка, .. - предыдущая по главенству папка
3. rm - remove - удаление, используется с флагом -r (recursive)
4. cd - change directory, можно использовать без флагов
5. cp - copy paste, вводится сначала cp, затем то, что ты копируешь, и потом то, куда копируешь(cp ~/feat ~/test)
6. mv - move - перемещение чего либо, куда либо(работает по принципу cp)
7. переименовать папку/файл можно с помощью команды nano, введя после этого точное имя папки/файла(откроется редактор), после чего при сохранении изменений в редакторе можно будет переименовать файл
8. переместить файл/папку можно с помощью команды mv(move) - mv ~/feat ~/test
9. флаг -R работает так же как и -r
10. -r (recursive) это поведение функции, при котором она вызывает сама себя, например rm -r
11.
12. git init это команда для создания нового репозитория
13.Команда git branch создает ветку. Например: git branch  "название ветки"
14.поменять ветку можно с помощью команды git checkout "имя ветки"
15.
16. Git restore staged (git r16. Git restore staged (git rm)– это команда Git, которая позволяет вернуть изменения из области подготовленных к коммиту файлов (staged) обратно в рабочую директорию.
17. команда git revert создаёт новый коммит, который вносит изменения, противоположные указанному коммиту, по существу отменяя его.
